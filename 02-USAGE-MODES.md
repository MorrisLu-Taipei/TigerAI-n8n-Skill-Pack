# 使用模式總覽

> 🌐 [English](02-USAGE-MODES.en.md) | **繁體中文**

> 本 Skill Pack 支援 **三種使用方式**。使用者可依熟悉度任選。AI 收到觸發詞時會自動切換對應模式。
>
> **完全不用學語法**——整個 Pack 接受純自然語言中文（或英文）描述，AI 自己處理結構化解析。

---

## 模式 1：Cookbook 照抄模式（最快）

**適合誰**：已知道自己要什麼流程，想快速產出。

**使用方式**：
1. 開 [`cookbook/00-INDEX.md`](cookbook/00-INDEX.md) 找最像的範例
2. 把該範例的「🌱 自然語言版」內容複製到 n8n 畫布的便利貼
3. 依需求改幾個值（你的 channel 名 / API URL / 寄件人）
4. 呼叫 AI → 產出完整 workflow

**觸發詞**：使用者直接送出便利貼，無需特別說。

> 📐 cookbook 每個範例都附「進階 DSL 嚴謹寫法」（折疊區塊），給工程師或想嚴格規格化的使用者選擇用。**不熟技術完全不用看，自然語言版已足夠**。

---

## 模式 2：問答模式（最友善）⭐

**適合誰**：不確定怎麼描述需求、第一次用 n8n、需要引導的使用者。

**使用方式**：
使用者對 AI 說：
> 「**啟用問答模式**」
> 「我要做一個自動化流程，請帶我寫」
> 「Q&A mode」

AI 會分**5 個階段**循序提問，每階段一次只問 1–3 題：

| 階段 | 提問範例 |
|---|---|
| 1. 觸發 | 這個流程什麼時候要跑？(時間到 / 收到外部事件 / 手動 / 表單) |
| 2. 輸入 | 觸發時會帶哪些資料？欄位有哪些？ |
| 3. 步驟 | 收到資料後要做什麼？(可逐步加，AI 隨時複述) |
| 4. 輸出 | 處理完要送去哪？(Slack / Email / DB / API) |
| 5. 錯誤 | 失敗時怎麼處理？(忽略 / 通知人 / 重試) |

每階段結束 AI 會**回顯**目前已收集到的 Layer 1 sticky note 草稿。
全部問完後一次產出三層結構 workflow。

**詳細邏輯**：見 [`skills/tigerai/tigerai-qa-mode/SKILL.md`](skills/tigerai/tigerai-qa-mode/SKILL.md)

## 模式 3：範例查詢模式（最能學習）

**適合誰**：知道大概想做什麼，但想先看看別人怎麼做、再決定自己的版本。

**使用方式**：
使用者對 AI 說：
> 「**範例查詢模式**：找跟 X 相關的範例」
> 「有沒有人做過 X 的 workflow？」
> 「show me examples for X」

AI 會：
1. 從 `reference-workflows/`（2,061 個語料）+ `cookbook/` 搜尋相似度最高的 3–5 個
2. 對每個範例**用白話**說明：
   - 這個 workflow 解決什麼問題
   - 用了哪些節點、為什麼
   - 與你的需求有哪些差異
3. 詢問使用者：「要直接用某個範例改？還是要我幫你產一個全新的？」

**詳細邏輯**：見 [`skills/tigerai/tigerai-example-finder/SKILL.md`](skills/tigerai/tigerai-example-finder/SKILL.md)

## 三種模式比較

| 模式 | 速度 | 學習曲線 | 客製化 | 適合場景 |
|---|---|---|---|---|
| Cookbook 照抄 | ⚡⚡⚡ | 低 | 中 | 重複性高的標準流程 |
| 問答模式 | ⚡⚡ | 最低 | 高 | 沒寫過、需要引導 |
| 範例查詢 | ⚡ | 中 | 最高 | 想參考最佳實踐再決定 |

---

## 模式可組合使用

實際情境常是：

```
使用者 → 範例查詢「找跟客服分流相關的」
       → 看完後說「我要類似 cookbook/04 但加上資料庫紀錄」
       → 啟動問答模式補齊缺漏 (DB schema、credential)
       → AI 產出 workflow
```

AI 會記住整段脈絡，不需重新解釋需求。
